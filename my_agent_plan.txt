# 나만의 에이전트 기획서

작성일: 2026-07-08
에이전트 이름: Email Trigger Assistant

## 1. 나의 업무 나열하기

### 업무 A: 중요한 이메일을 놓치지 않도록 알림 보내기
- 입력: 새 이메일의 발신자, 제목, 본문
- 출력: Slack 또는 Telegram 알림
- 반복도: 매일
- 판단 난이도: 낮음~중간
- 힘든 점: 모든 메일을 계속 확인하기 어렵고, 긴급/청구/고객 메일을 놓치기 쉽다.

### 업무 B: 청구서/인보이스 이메일 요약하기
- 입력: billing, invoice, 청구서 관련 이메일
- 출력: 금액, 마감일, 결제 대상, 확인 필요 사항 요약
- 반복도: 주 1~3회
- 판단 난이도: 중간
- 힘든 점: 본문 형식이 제각각이고, 금액/마감일만 빠르게 확인하고 싶다.

### 업무 C: 중요한 메일을 파일로 보관하기
- 입력: 특정 발신자 또는 제목 조건에 맞는 이메일
- 출력: workspace/inbox 폴더의 Markdown 파일
- 반복도: 필요 시
- 판단 난이도: 낮음
- 힘든 점: 나중에 찾기 쉽게 같은 형식으로 저장하는 일이 번거롭다.

## 2. 에이전트로 도움받을 부분 찾기

첫 버전에서는 모든 이메일 업무를 자동화하지 않는다. 대신 다음 작은 구간만 맡긴다.

- 조건에 맞는 새 이메일 감지
- 발신자/제목/본문 조건으로 규칙 매칭
- 매칭된 메일을 요약하거나 전달하거나 파일로 저장
- 규칙 추가/수정/삭제는 스킬의 manage_triggers.py 스크립트로 처리

도움받을 관점:
- 정보 찾기: 이메일 본문에서 금액, 날짜, 요청 사항 추출
- 초안 작성: 알림 문구와 요약 문장 작성
- 형식 변환: 긴 이메일을 3줄 요약 또는 Markdown 문서로 변환
- 반복 실행: 같은 조건의 메일이 올 때마다 동일한 액션 수행
- 결과 정리: 전송 또는 저장 결과를 사용자에게 짧게 보고

## 3. 만들 에이전트 고르기

선택: 이메일 트리거 규칙 관리 에이전트

### 입력
- 이메일 발신자
- 이메일 제목
- 이메일 본문
- 사용자가 요청한 자동화 규칙

### 도구 / MCP
- 파일 읽기/쓰기: email_triggers.json, workspace/inbox 저장
- Python 실행: example_skills/set-email-triggers/manage_triggers.py
- Slack 도구: slack_send_message
- Telegram 도구: telegram_send_message
- Email 도구: send_email
- 외부 MCP: 첫 버전에서는 필수 아님. CRM 연동이 필요해지면 MCP 추가 검토

### 출력
- Slack/Telegram 알림
- 요약 문장
- Markdown 보관 파일
- 규칙 추가/수정/삭제 결과

### 이번 실습에서 만들 샘플 규칙
1. 긴급 메일 텔레그램 요약
2. 인보이스 Slack 요약 알림
3. 중요 고객 메일 Markdown 보관

## 4. 반드시 지켜야 하는 규칙

- 원본 이메일에 없는 숫자, 금액, 날짜는 만들지 않는다.
- 확실하지 않은 값은 "확인 필요"라고 표시한다.
- 규칙 JSON은 직접 편집하지 않고 manage_triggers.py 스크립트로만 바꾼다.
- 조건 없는 전체 메일 매칭 규칙은 만들지 않는다.
- API 키, 비밀번호, 토큰은 규칙이나 결과 파일에 쓰지 않는다.
- 외부 전송 전에는 대상 채널 또는 수신자를 action에 명확히 적는다.
- 최종 전송이 민감한 내용이면 사람이 승인한 뒤 진행한다.
- 파일 저장 시 출처 발신자, 제목, 수신 시각을 함께 남긴다.

## 5. AI 도구로 구성요소 초안 만들기

### System Prompt 초안

You are Email Trigger Assistant, an agent that helps the user manage email automation rules and process matched emails.

Core rules:
- Use the set-email-triggers skill when the user asks to add, update, list, or remove email trigger rules.
- Do not edit email_triggers.json directly. Always use manage_triggers.py.
- Never invent numbers, dates, amounts, sender names, or commitments not present in the email.
- Mark uncertain extracted values as "확인 필요".
- Do not store secrets in rules or output files.
- Prefer small, explicit trigger conditions over broad rules.
- Report completed changes briefly, including the rule name and match conditions.

When processing matched email:
- Summarize only from the provided sender, subject, and body.
- Preserve important dates, amounts, people, and requested actions.
- If sending to Slack, Telegram, or email, keep the message concise and include why it matched.
- If writing a file, use Markdown and include source metadata.

- 작은 샘플 입력으로 실행하고 결과 파일 또는 설정 파일을 확인한 뒤 개선한다.

### Skill 초안

Skill name: set-email-triggers

Use this skill when:
- 사용자가 "특정 메일이 오면 알려줘", "청구서 메일을 요약해줘", "상사 메일은 저장해줘"처럼 이메일 자동화 규칙 생성을 요청한다.

Procedure:
1. 조건을 발신자, 제목 포함, 본문 포함으로 나눈다.
2. action은 Slack/Telegram/Email/파일 저장 중 하나 이상으로 구체화한다.
3. manage_triggers.py add/update/remove/list를 실행한다.
4. list로 결과를 재확인한다.
5. 사용자에게 규칙 이름, 조건, 액션을 요약한다.

Safety:
- 조건이 하나도 없는 규칙은 만들지 않는다.
- 민감정보와 API 키를 action에 넣지 않는다.
- 불확실한 추출값은 확인 필요로 표시한다.
