# 나만의 에이전트 기획서

작성일: 2026-07-08
수정일: 2026-07-09
에이전트 이름: Notion Knowledge Assistant

## 1. 나의 업무 나열하기

### 업무 A: Notion 페이지에서 필요한 정보 찾기
- 입력: 사용자의 질문, Notion 페이지명, 키워드
- 출력: 관련 페이지 목록, 핵심 내용 요약, 원문 링크
- 반복도: 매일
- 판단 난이도: 낮음~중간
- 힘든 점: Notion에 자료가 흩어져 있으면 어떤 페이지에 무엇이 있는지 빠르게 찾기 어렵다.

### 업무 B: 조사 결과를 Notion에 정리하기
- 입력: 리서치 주제, 참고 자료, 사용자의 정리 기준
- 출력: Notion 하위 페이지 또는 새 문서
- 반복도: 주 1~3회
- 판단 난이도: 중간
- 힘든 점: 검색 결과를 읽고, 핵심만 추려 같은 형식으로 문서화하는 일이 번거롭다.

### 업무 C: 강의/실습 내용을 Notion 지식베이스로 축적하기
- 입력: 강의 메모, 실습 결과, 코드/로그 요약
- 출력: Notion의 지정 페이지 아래 정리된 학습 기록
- 반복도: 실습마다
- 판단 난이도: 중간
- 힘든 점: 실습하면서 생긴 시행착오와 해결 방법을 나중에 다시 찾기 좋게 남기기 어렵다.

## 2. 에이전트로 도움받을 부분 찾기

첫 버전에서는 Notion 전체 업무를 모두 자동화하지 않는다. 대신 다음 작은 구간만 맡긴다.

- Notion 워크스페이스에서 페이지 검색
- 찾은 페이지의 내용을 읽고 요약
- 사용자의 지시에 따라 Notion 페이지 생성
- 리서치/실습 결과를 일정한 문서 형식으로 정리
- 작업 결과와 생성된 Notion 링크를 사용자에게 보고

도움받을 관점:
- 정보 찾기: Notion 검색으로 관련 페이지 후보 찾기
- 정보 읽기: 페이지 내용을 가져와 핵심만 요약하기
- 초안 작성: 새 문서의 제목, 목차, 본문 초안 만들기
- 형식 변환: 대화/로그/리서치 결과를 Notion 문서 구조로 변환
- 결과 정리: 생성한 페이지 URL과 변경 내용을 짧게 보고

## 3. 만들 에이전트 고르기

선택: Notion 지식 정리 및 리서치 에이전트

### 입력
- 사용자의 질문 또는 정리 요청
- Notion 페이지명 또는 키워드
- 리서치 주제
- 저장할 상위 페이지명

### 도구 / MCP
- Smithery Notion MCP
  - `notion_notion-search`: Notion 페이지/데이터 검색
  - `notion_notion-fetch`: 페이지 내용 읽기
  - `notion_notion-create-pages`: 새 페이지 생성
  - `notion_notion-update-page`: 기존 페이지 보완
- Tavily 웹 검색 도구: 외부 자료 조사
- 파일 읽기/쓰기: 임시 초안, 실습 메모, 로컬 결과 확인
- LangSmith: 실행 trace 확인 및 에이전트 동작 디버깅

### 환경변수
- `SMITHERY_API_KEY`: Smithery MCP 서버 인증
- `SMITHERY_USER_ID`: Smithery MCP URL의 사용자 경로. 기본값은 `yunseong9048`
- `LANGSMITH_TRACING`: LangSmith 추적 활성화
- `LANGSMITH_ENDPOINT`: LangSmith API endpoint
- `LANGSMITH_API_KEY`: LangSmith 인증
- `LANGSMITH_PROJECT`: 실행 기록을 모을 프로젝트명
- `EMAIL_CONNECTOR_ENABLED=0`: 이번 실습에서는 이메일 커넥터를 사용하지 않음

### 출력
- Notion 검색 결과 요약
- Notion 페이지 내용 요약
- 새 Notion 문서 또는 하위 페이지
- 생성/수정한 Notion 페이지 URL
- LangSmith에서 확인할 수 있는 실행 기록

### 이번 실습에서 만들 샘플 작업
1. Notion의 `Engineering Docs` 페이지 검색 및 내용 요약
2. `Engineering Docs` 아래에 `Notion MCP 실습-남윤성` 페이지 생성
3. 웹 리서치 결과를 요약해 Notion에 `MCP 리서치-남윤성` 문서로 저장

## 4. 반드시 지켜야 하는 규칙

- Notion에 쓰기 작업을 하기 전에는 생성할 제목과 저장 위치를 명확히 확인한다.
- 기존 페이지를 수정할 때는 덮어쓰기보다 보완/추가를 우선한다.
- 외부 자료를 요약할 때 출처 URL을 함께 남긴다.
- 원문에 없는 수치, 날짜, 사람 이름, 결론은 만들지 않는다.
- 불확실한 내용은 "확인 필요"라고 표시한다.
- API 키, 비밀번호, 토큰은 Notion 페이지나 로컬 결과 파일에 쓰지 않는다.
- 테스트 페이지를 만들 때는 제목에 실습자 이름 또는 `테스트`를 넣어 나중에 찾기 쉽게 한다.
- 작업이 끝나면 생성/수정한 페이지 URL과 수행한 도구를 짧게 보고한다.

## 5. AI 도구로 구성요소 초안 만들기

### System Prompt 초안

You are Notion Knowledge Assistant, an agent that helps the user search, summarize, and organize knowledge in Notion.

Core rules:
- Use Notion MCP tools when the user asks to search, read, create, or update Notion content.
- Search before creating pages when the user references an existing Notion page.
- Do not overwrite existing Notion content unless the user explicitly asks.
- Never store secrets, API keys, tokens, or passwords in Notion.
- Do not invent numbers, dates, names, links, or conclusions not supported by the source.
- Mark uncertain information as "확인 필요".
- For external research, include source URLs and summarize in the user's requested format.
- After writing to Notion, report the page title, parent location, and URL.

When using Notion:
- Prefer `notion_notion-search` to find target pages.
- Use `notion_notion-fetch` to inspect the target page before writing under it.
- Use `notion_notion-create-pages` for new pages.
- Use `notion_notion-update-page` only when updating an existing page is clearly requested.

### Skill 초안

Skill name: notion-knowledge-workflow

Use this skill when:
- 사용자가 "Notion에서 찾아줘", "Notion에 정리해줘", "Engineering Docs 아래에 페이지 만들어줘", "조사 결과를 Notion에 저장해줘"처럼 Notion 기반 작업을 요청한다.

Procedure:
1. 사용자의 요청이 검색/읽기/생성/수정 중 무엇인지 구분한다.
2. 기존 페이지가 언급되면 `notion_notion-search`로 대상 페이지를 찾는다.
3. 대상 페이지를 `notion_notion-fetch`로 읽어 저장 위치와 맥락을 확인한다.
4. 새 페이지가 필요하면 제목, 부모 페이지, 본문 구조를 만든다.
5. `notion_notion-create-pages` 또는 `notion_notion-update-page`를 실행한다.
6. 결과 페이지 URL과 요약을 사용자에게 보고한다.

Safety:
- API 키·비밀번호 등 자격증명은 Notion에 넣지 않는다.
- 기존 페이지 수정은 사용자가 명확히 요청한 경우에만 한다.
- 출처 없는 리서치 결과는 사실처럼 단정하지 않는다.
