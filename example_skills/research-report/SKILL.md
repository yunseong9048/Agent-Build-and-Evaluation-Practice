# 리서치 보고서 스킬

특정 주제(기술 동향/시장/정책/산업/사용자 실태)를 웹에서 조사해 **의사결정용 근거 기반 보고서**를 만든다. web-research 스킬이 "빠른 요약"이라면, 이 스킬은 **여러 출처 교차검증 + 정식 보고서 산출물**이 목적이다.

## 언제 쓰나

- "출처 포함해서", "최신 자료 기반", "비교 분석", "보고서 형태" 키워드가 등장할 때
- 기술 동향·시장 규모·정책·경쟁 구도 등 외부 사실을 정리해 문서로 남겨야 할 때
- 단일 기사 요약이 아니라 다수 출처를 종합한 분석이 필요할 때

## 절차

1. **요구사항 확정**: 주제 / 범위(지역·기간·대상) / 조사 축(개념·사례·한계·전망 등) / 분량·형식을 확인한다. 불명확하면 최소 한 가지만 되묻고 나머지는 기본값(최근 1~3년, 3~5페이지 분량)으로 진행한다.

2. **조사 질문 정의**: 주제를 답해야 할 **핵심 질문 3~5개**로 분해한다. 필요하면 `write_todos`로 남긴다.

3. **자료 수집 (웹 검색 필수)**: 각 조사 질문마다 키워드를 바꿔 웹 검색 커넥터를 **여러 번** 호출한다. 학술/공식/공공/시장조사 출처를 우선한다(`references/source-priority.md`). 자료마다 제목/기관/발행일/유형/URL/핵심내용을 기록한다.

4. **신뢰도 평가 & 교차검증**: 출처 권위·발행일·조사방법·이해관계를 본다. **핵심 주장은 서로 다른 출처 2개 이상**으로 확인한다. 하나뿐이면 `[단일출처]`로 표시한다. 상충하는 자료는 양쪽을 함께 제시한다.

5. **분석**: 사실과 해석을 분리한다. 정량/정성 자료의 균형을 맞춘다. 편향(기업/국가/기술)과 자료 한계(표본·기간·지역)를 명시한다.

6. **보고서 작성**: `templates/report.md` 형식을 따른다. **핵심 요약을 맨 앞에** 두고, 모든 주장을 출처에 연결한다. `write_file`로 `workspace/`에 마크다운으로 저장하고 경로를 알린다.

7. **검토** (`references/citation-rules.md` 체크리스트): 허위 출처가 없는가 · 모든 주장에 근거가 있는가 · 발행일이 있는가 · 사실/해석이 분리됐는가 · 한계가 명시됐는가.

## 반드시 지키는 규칙 (요약)

- **존재하지 않는 논문·통계·URL·인용을 절대 만들지 않는다.** 불확실하면 `[확인 필요]`.
- **모든 주장·수치를 출처에 연결한다.** 링크·기관·발행일을 함께 남긴다.
- **단일 출처 주장은 `[단일출처]`로 표시**하고 확정적으로 서술하지 않는다.
- **사실과 해석을 분리**하고, 편향·한계를 명시한다.
- **한국어 실무 보고서 문체.** 과장 수식어 없이 사실 위주. 고유명사는 원문 표기 유지.

자세한 규칙·출처 우선순위·템플릿은 `references/`, `templates/report.md` 참조.
